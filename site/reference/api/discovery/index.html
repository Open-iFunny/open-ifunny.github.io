    🧭 Discovery¶

    Explore compilations, weekly digests, and other content-discovery feeds.

    POST /explore/compilation/{id} — Get Explore Compilation¶

    Fetch a page of an explore compilation feed (curated content shelves). Method is POST (matching upstream behaviour) despite semantically being a read.

    Base URL: https://api.ifunny.mobi/v4
    Auth: BearerAuth + ProjectId

    Path parameters¶

    Name Type Required Description
    id String yes Compilation ID (see EXPLORE_COMPILATION_IDS in client types).

    Query parameters¶

    Name Type Required Description
    limit Number no
    next String no
    prev String no

    Query — combined type

  "next"?: "string",
  "prev"?: "string"

    GET /digest_groups — Get Weekly Digest Groups¶

    Paginated list of weekly digests. Optional comments / contents flags inline extended detail data in each digest entry.

    Base URL: https://api.ifunny.mobi/v4
    Auth: BearerAuth + ProjectId

    Query parameters¶

    Name Type Required Description
    limit Number no
    next String no
    comments Number no 1 to include comment detail data. — One of: 0, 1
    contents Number no 1 to include content detail data. — One of: 0, 1

    Query — combined type

    // GetDigestGroupsQuery
{
  "limit"?: "integer",
  "next"?: "string",
  "comments"?: "enum(0, 1)",
  "contents"?: "enum(0, 1)"
}

    interface GetDigestGroupsQuery {
  limit?: number;
  next?: string;
  comments?: 0 | 1;
  contents?: 0 | 1;
}

    type GetDigestGroupsQuery struct {
    Limit *int `query:"limit,omitempty"`
    Next *string `query:"next,omitempty"`
    Comments *getDigestGroupsQueryCommentsKind `query:"comments,omitempty"`
    Contents *getDigestGroupsQueryContentsKind `query:"contents,omitempty"`
}
