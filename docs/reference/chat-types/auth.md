# 🤝 Auth

To use iFunny Chats, you must establish an authenticated WebSocket connection to the WAMP realm. This page covers the authentication handshake and bearer token credential flow.

## WebSocket Connection

Connect to the iFunny chat WAMP server:

```
wss://chat.ifunny.co/chat
```

Set the WebSocket subprotocol header to:

```
Sec-WebSocket-Protocol: wamp.json
```

> Note: Some WAMP clients default to `wamp.2.json`. The iFunny chat gateway accepts both, but ensure your client is configured to send `wamp.json`.

## Bearer Token

Authentication uses the OAuth2 bearer token. See [oauth2.md](../api/oauth2.md#op-loginorrefresh) to obtain a bearer token before connecting.

## WAMP Handshake

The WAMP handshake consists of four message phases:

### 1. Hello

When your client connects, send a HELLO message to announce yourself to the server:

```json
[
  1,
  "co.fun.chat.ifunny",
  {
    "authmethods": ["ticket"],
    "roles": {
      "subscriber": {},
      "publisher": {}
    }
  }
]
```

**Message fields:**
- `[0]` = `1` (WAMP HELLO message ID)
- `[1]` = `"co.fun.chat.ifunny"` (Realm URI)
- `[2].authmethods` = `["ticket"]` (Accept ticket-based auth)
- `[2].roles` = Roles your client will use (subscriber, publisher)

### 2. Challenge

The server responds with a CHALLENGE message:

```json
[
  4,
  "ticket",
  {}
]
```

**Message fields:**
- `[0]` = `4` (WAMP CHALLENGE message ID)
- `[1]` = `"ticket"` (Confirms ticket auth method)
- `[2]` = `{}` (Empty details object; required by WAMP spec)

### 3. Authenticate

Reply with your bearer token in an AUTHENTICATE message:

```json
[
  5,
  "your_bearer_token_here",
  {}
]
```

**Message fields:**
- `[0]` = `5` (WAMP AUTHENTICATE message ID)
- `[1]` = Your OAuth2 bearer token (send as-is; no signature computation needed)
- `[2]` = `{}` (Empty details object; required by WAMP spec)

### 4. Welcome

The server responds with a WELCOME message containing your session ID and user metadata:

```typescript
[
  2,
  number,           // Session ID (generated by server)
  {
    "authid": string,          // Your iFunny user ID
    "authrole": "user",
    "authmethod": "ticket",
    "authprovider": "users",
    "attributes": {
      "nick": string           // Your iFunny nickname
    },
    "agent": "co.fun.chat",
    "roles": {
      "broker": {
        "features": {
          "pattern_based_subscription": true,
          "publisher_exclusion": true,
          "publisher_identification": true,
          "subscriber_blackwhite_listing": true,
          "session_meta_api": true,
          "subscription_meta_api": true
        }
      },
      "dealer": {
        "features": {
          "call_canceling": true,
          "call_timeout": true,
          "caller_identification": true,
          "pattern_based_registration": true,
          "progressive_call_results": true,
          "shared_registration": true,
          "session_meta_api": true,
          "registration_meta_api": true,
          "testament_meta_api": true
        }
      }
    }
  }
]
```

**Message fields:**
- `[0]` = `2` (WAMP WELCOME message ID)
- `[1]` = Session ID — store this; it identifies your connection for the lifetime of the session
- `[2].authid` — Your iFunny numeric user ID; use this to build topic URIs like `co.fun.chat.user.{authid}.chats`
- `[2].attributes.nick` — Your iFunny display name
- `[2].roles` — Broker and dealer features advertised by the server

## Next Steps

Once authenticated, you can:
- **Subscribe** to topics (e.g., `co.fun.chat.user.{authid}.chats`) to receive real-time events
- **Call** procedures (e.g., `co.fun.chat.create_channel`) to perform actions
- **Publish** to topics to send messages to channels

See the other Chat Types pages for specific operations:
- [Chats + DMs](chats-dms.md) — Create channels, send/receive messages, list message history
- [Subscriptions + Events](subscriptions-events.md) — Real-time event streams
- [WAMP](wamp.md) — Invites, moderation, and advanced protocol details
